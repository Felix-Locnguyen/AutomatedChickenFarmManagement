KẾ HOẠCH KỸ THUẬT: HỆ THỐNG QUẢN LÝ TRANG TRẠI GÀ THÔNG MINH
  (BACKEND)

  1. KIẾN TRÚC HỆ THỐNG (SYSTEM ARCHITECTURE)
  Dự án được xây dựng theo kiến trúc Modular Monolith sử dụng mô
  hình Application Factory:
   * Application Factory (create_app): Giúp tách biệt việc khởi tạo
     ứng dụng khỏi việc chạy ứng dụng. Điều này cho phép dễ dàng
     thay đổi cấu hình (Dev/Test/Prod) và tránh lỗi vòng lặp phụ
     thuộc (circular imports).
   * Flask Blueprints: Chia nhỏ hệ thống thành các module độc lập
     (Auth, Coop, Device, Dashboard). Mỗi module quản lý một nhóm
     tính năng riêng, giúp code sạch và dễ bảo trì.

  3. THIẾT KẾ CƠ SỞ DỮ LIỆU (DATABASE SCHEMA)
  Hệ thống quản lý 4 nhóm dữ liệu chính:
   1. Identity (User): Lưu trữ định danh và phân quyền
      (Admin/Worker).
   2. Infrastructure (Coop & Device): Quản lý chuồng trại và các
      thiết bị IOT (Cảm biến, máy bơm, quạt).
   3. Operation (Environment & FeedSchedule): Lưu trữ dữ liệu thực
      tế từ cảm biến và lịch trình vận hành.
   4. Monitoring (Alert): Nhật ký cảnh báo khi hệ thống phát hiện
      bất thường.

  4. LỘ TRÌNH TRIỂN KHAI (ROADMAP)

  Giai đoạn 1: Thiết lập hạ tầng cốt lõi (Foundation)
   * Mục tiêu: Tạo "trái tim" cho ứng dụng.
   * Công việc:
       * Hoàn thiện app.py theo mô hình Application Factory.
       * Cấu hình kết nối Database và các Extensions (db, jwt,
         cors).
       * Đăng ký hệ thống Blueprint tổng cho toàn bộ các route API.

  Giai đoạn 2: Xác thực và Bảo mật (Security First)
   * Mục tiêu: Đảm bảo chỉ người dùng hợp lệ mới có thể xem dữ liệu
     và điều khiển trang trại.
   * Công việc: Triển khai các Endpoint: /api/auth/register,
     /api/auth/login, /api/auth/me.

  Giai đoạn 3: Nghiệp vụ Quản lý IOT (Business Logic)
   * Mục tiêu: Quản lý chuồng trại và thiết bị.
   * Công việc:
       * Xây dựng API CRUD cho Chuồng (Coops) và Thiết bị
         (Devices).
       * Triển khai tính năng Toggle Device (Bật/Tắt thiết bị từ xa
         qua API).

  Giai đoạn 4: Dữ liệu lớn và Cảnh báo (Data & Monitoring)
   * Mục tiêu: Thu thập dữ liệu cảm biến và tự động cảnh báo.
   * Công việc:
       * Xây dựng API nhận dữ liệu từ cảm biến (/api/environment).
       * Xây dựng Logic kiểm tra ngưỡng (Threshold) để tự động tạo
         Alert.
       * API lấy dữ liệu lịch sử để vẽ biểu đồ Dashboard.

  Giai đoạn 5: Tích hợp và Hoàn thiện (Integration)
   * Mục tiêu: Kết nối hoàn chỉnh Backend với Frontend.
   * Công việc:
       * Nạp dữ liệu mẫu bằng seed.py.
       * Kiểm tra tính tương thích với các trang HTML trong thư mục
         static/.

  5. TIÊU CHUẨN VẬN HÀNH (STANDARDS)
   * API Response: Luôn trả về định dạng { "status":
     "success/error", "data": ..., "message": ... }.
   * Status Codes: Sử dụng đúng mã lỗi (200: OK, 201: Created, 401:
     Unauthorized, 404: Not Found, 500: Server Error).
   * Logging: Ghi lại các hoạt động quan trọng như đăng nhập thất
     bại hoặc thiết bị mất kết nối.

  > Nhiệm vụ 1: Cập nhật trang coop-detail.html — Đồng bộ dữ liệu từ API & xử lý logic tương tác.
  >
  > 1. Các trường dữ liệu cần lấy từ API (Sử dụng coopsAPI và environmentAPI trong api.js):
  > - Thông tin chuồng: name, current_count, area, location, status.
  > - Thông tin ngưỡng: temp_min, temp_max, humidity_min, humidity_max.
  > - Dữ liệu môi trường (mới nhất): temperature, humidity, feed_level, water_level.
  > - Thiết bị: Danh sách devices (tên, loại, trạng thái is_active).
  >
  > 2. Logic xử lý dữ liệu:
  > - Lấy ID từ URL: Sử dụng JavaScript để trích xuất id từ query string hoặc path.
  > - Khởi tạo: Gọi coopsAPI.getOne(id) và coopsAPI.getEnvironment(id) khi trang load.
  > - Render: Viết một hàm renderCoopDetail(data) để đổ dữ liệu vào các thẻ HTML (Info cards, tiêu đề).
  >
  > 3. Logic tương tác (Cập nhật Dataset):
  > - Đổi tên/Vị trí/Số lượng: Khi người dùng nhấn Save, gọi coopsAPI.update(id, updatedData). Nếu thành công, cập nhật
  lại tiêu đề trang Chi tiết <name> và các thẻ thông tin mà không reload trang.
  > - Điều khiển thiết bị: Khi nhấn Toggle thiết bị, gọi devicesAPI.toggle(deviceId). Cập nhật icon/màu sắc trạng thái
  ngay khi API trả về kết quả thành công.
  >
  > 4. Yêu cầu kỹ thuật:
  > - Sử dụng đúng các hàm đã định nghĩa trong static/js/api.js.
  Tại sao bản này tốt hơn?
   1. Đúng tên hàm: Nhắc đích danh coopsAPI.getOne hay devicesAPI.toggle giúp code không bị sai lệch.
   2. Đúng kiến trúc: Phù hợp với cách Flask đang chạy và jQuery đang xử lý.
   3. Thực tế: Tập trung vào việc gọi API và thay đổi DOM (đúng sở trường của jQuery).
